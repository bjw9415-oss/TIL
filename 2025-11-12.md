## 오늘 배운 것들

- 브랜치(Branch)
```
브랜치는 독립된 작업 공간으로 하나의 프로젝트에서 동시에 여러 작업을 하기 위해 가지(Branch)를 만들어서 관리한다.

```

이렇게 하는 이유는 main은 사용자들이 지켜보고 있는 장소 이므로 아예 분리된 장소에서 작업을 해야 한다. 

브랜치도 혼자 작업 할때는 이름을 막 써도 상관없으나 결국 협업을 해야하기 때문에 누구나 알아볼 수 있는 네이밍이 필요하다~ 

브랜치 하는 방법
1. 메인을 체크아웃 한다.
2. 소스트리에서 브랜치 버튼을 누른다.
3. 해야 되는 작업에 맞게 이름을 작성한다.
4. 브랜치된 곳을 체크아웃하고 vs코드에서 작업한다~

- 병합(merge) 

``` 
브랜치와 브랜치를 합쳐 주는것을 병합이라고 한다.
```

병합하는 순서
1. 소스트리에서 병합 할 곳을 체크아웃 한다.
2. 메인과 병합한다.
3. 충돌이 나게되면 vs코드에서 확인 후 수정하고 저장한다.
4. 에드,커밋,푸쉬 후 메인을 체크아웃한다.
5. 병합 한 곳을 다시 병합한다.
6. 푸쉬하고 원격으로 github 에서 확인한다.

또 다른 방법
1. 소스트리에서 메인을 브랜치 하고 내용을 작성한다.
2. 에드,커밋,푸쉬 하고 github에 들어간다.
3. 들어가서 pull requests 를 같이 협업하는 사람에게 request를 요청? 하고 승인하면 github 내에서 병합이 가능하다.
