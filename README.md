## 오늘 배운것들 
- PUSH,PULL,CLONE 이론 및 실습

```
PUSH는 기본적으로 로컬인 내 컴퓨터에서 원격저장소인 GITHUB 으로 
작업했던 폴더 파일을 보내는 것.
PULL은 반대로 GITHUB 에서 내 컴퓨터로 폴터 파일을 보낸다.

CLONE 은 만약 내 컴퓨터나 다른 작업장소에서 작업을 하려고 할때 파일이나 폴더가 없다면 GITHUB에서 통째로 다운받는 것.
```

## 어제 배운것 요약 
```
GIT 은 분산버전 관리 시스템이다. 말 그대로 여러가지 포인트 지점을 만들어서 관리할 수 있는 좋은 시스템.

GITHUB 의 장점: 협업,백업이 용이하다. 깃으로 관리하는 프로젝트를 올려둘 수 있는 호스팅 사이트


ADD는 커밋을 하기전 스테이지에 올리는 작업으로 마치 장을 보기위해 
장바구니에 리스트를 올리는것과 같다. 

COMMIT 은 ADD 된 폴더나 파일을 저장하는 작업이다. 
```
