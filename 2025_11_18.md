# 20205.11.18 TIL

## 어제 배운것들

-Tailwind CSS

```
Utility-First CSS Framework로 분류되는 스타일 프레임워크
```

### Utility-first 방식의 장점

- CSS 작성 최소화
- 클래스 조합만으로 즉시 스타일 확인 가능
- 컴포넌트 일관성 유지
- 디자인 시스템 구축 용이
- 유지보수 쉬움

Tailwind CSS의 장점

Tailwind CSS는 Utility-first 방식을 기반으로 하여 다음과 같은 장점을 제공한다.

### **1) 높은 생산성**

- CSS 파일을 작성하지 않아도 HTML에서 바로 스타일을 구성할 수 있다.
- 빠른 프로토타이핑 및 레이아웃 구현이 가능하다.

### **2) 디자인 일관성 유지**

- Tailwind가 제공하는 미리 정의된 spacing, color, typography scale을 사용하면
  프로젝트 전체에서 디자인 시스템이 자연스럽게 일관성을 갖는다.

### **3) 커스터마이징 자유도**

- `tailwind.config.js`를 통해 색상, 간격, 폰트 등 거의 모든 요소를 원하는 값으로 확장할 수 있다.
- Bootstrap처럼 특정 디자인 틀에 묶이지 않는다.

### **4) 반응형 디자인이 간단함**

- `sm:`, `md:`, `lg:` 와 같은 접두어를 활용해 반응형 스타일을 간단하게 작성할 수 있다.
- 미디어쿼리를 직접 작성할 필요가 없다.

### **5) CSS 중복 최소화**

- 클래스를 조합해 사용하므로 개별 CSS 선택자나 모듈을 중복 생성할 필요가 없다.
- 빌드 환경에서는 실제 사용된 클래스만 추출되어 최종 CSS 파일 크기가 작다.

## 1.1 CDN 방식

HTML 문서의 `<head>` 안에 **CDN 스크립트 한 줄만 추가하면 바로 Tailwind를 사용할 수 있는 방식**이다.
<
script src="https://cdn.tailwindcss.com"></script>

- 설치 과정이 없음
- 바로 사용 가능
- 작은 프로젝트, 실습, 프로토타이핑에 적합
- config 파일 없이 기본 Tailwind 설정 사용
- 빌드 과정이 없어 파일 크기 최적화는 제한됨

## 1.2 CLI방식

Node.js 기반으로 Tailwind를 설치하고,

자신만의 커스텀 Tailwind 설정(tailwind.config.js)을 사용하는 방식이다.

- 실제 프로젝트에서 가장 많이 사용됨
- 커스텀 테마, 색상, spacing, 폰트 설정 가능
- Purge 기능을 통해 사용하지 않는 CSS 제거 → CSS 파일 크기 최소화
- 빌드 과정 필요 (Vite, Webpack, Next.js 등과 함께 작동)

### 느낀점

- 생각보다 더 간편하고 이용하는데 직관적이다.
- 공식문서를 필요할 때 마다 보게 되는 습관이 길러졌다.
